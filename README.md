- branch명 규칙  
```
feature/signup : 새로운 기능 추가할 때(기능별 1개씩만 signup1, signup2 => x)  
fix/브랜치명 : 버그 수정할 때  
refactor/브랜치명 : 결과물은 그대로지만 코드를 깔끔하게 정리할 때
```

- commit 메세지 규칙  
```
feat : 새로운 기능 추가, 기존의 기능을 요구 사항에 맞추어 수정  
fix : 기능에 대한 버그 수정  
chore : 패키지 매니저 수정, 그 외 기타 수정 ex) .gitignore  
test : 테스트 코드 추가/수정  
```
